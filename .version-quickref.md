# Version Management Quick Reference

## Common Commands

### Check Current Version
```bash
bundle exec fastlane show_version
```

### Set Version Manually
```bash
bundle exec fastlane set_version version:1.2.0
```

### Build for TestFlight

```bash
# Default (patch bump): 1.1.0 → 1.1.1
bundle exec fastlane beta

# Minor bump: 1.1.0 → 1.2.0
bundle exec fastlane beta bump_type:minor

# Major bump: 1.1.0 → 2.0.0
bundle exec fastlane beta bump_type:major

# No version bump
bundle exec fastlane beta skip_version_bump:true
```

### Validate Versions
```bash
bundle exec fastlane validate_versions
```

## GitHub Actions

### Manual Release
1. Go to: **Actions** → **Release** workflow
2. Click **"Run workflow"**
3. Select bump type: `patch` | `minor` | `major`
4. Click **"Run workflow"**

### Tag Release
```bash
git tag v1.2.0
git push origin v1.2.0
```

## Semantic Versioning

Format: **MAJOR.MINOR.PATCH** (e.g., 1.2.3)

- **MAJOR** (1.x.x → 2.x.x): Breaking changes
- **MINOR** (1.1.x → 1.2.x): New features
- **PATCH** (1.1.1 → 1.1.2): Bug fixes

## Files
- `.version` - Current version number
- `ListAll/ListAll.xcodeproj/project.pbxproj` - Xcode project (auto-updated)
- `documentation/version_management.md` - Full documentation

## Troubleshooting

**Version mismatch?**
```bash
bundle exec fastlane set_version version:1.1.0
```

**Can't find project?**
- Run from repository root (not fastlane directory)
