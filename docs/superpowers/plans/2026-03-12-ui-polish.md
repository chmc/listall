# UI Polish Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Apply brand-consistent visual polish across all platforms — card-based item rows, teal sidebar selection, watchOS left-aligned redesign, and accent color unification.

**Architecture:** Pure UI changes across 18 SwiftUI view files. No model/service/viewmodel changes. Each task modifies 1-3 view files, verified via screenshot comparison against HTML mockups in `.superpowers/brainstorm/57429-1773298544/`.

**Tech Stack:** SwiftUI, Theme.swift (iOS/macOS), `.accentColor` (watchOS), MCP tools for visual verification.

- [ ] **Step 7: Add press feedback (2f)**

**WARNING:** The current code uses `onTapGesture` (~line 121) for toggle, a separate `Button` for chevron (~line 126), `.contextMenu` via `.if()` (~line 143), and `.swipeActions` (~lines 163, 179). **Recommended approach:** Use `DragGesture(minimumDistance: 0)` for press detection instead of a wrapping Button:
```swift
@State private var isPressed = false

// On the card container:
.scaleEffect(isPressed ? 0.97 : 1.0)
.animation(.easeInOut(duration: 0.15), value: isPressed)
.simultaneousGesture(
    DragGesture(minimumDistance: 0)
        .onChanged { _ in isPressed = true }
        .onEnded { _ in isPressed = false }
)
```

This coexists with existing gestures. If it causes issues with swipe actions, skip press feedback for now — it's the lowest priority sub-change.
